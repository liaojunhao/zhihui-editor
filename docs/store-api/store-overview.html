Store

Store 是您将要使用到的基本数据模型对象，它有%90的API方法都来自于 Zhihui Editor

import { createStore } from 'zhihui-editor/lib/model/store';

const store = createStore({
  token: 'YOUR_TOKEN' // 联系我们智绘助手获取
});

画布​

store.clear()​

清除画布上所有元素。

store.clear(); // 它将删除所有数据但保留撤消历史记录

// 启动深度清理
store.clear({ keepHistory: true }); // 它将删除所有数据并重置撤消历史记录

store.setSize(width, height)​

设置工作区域的尺寸。

store.setSize(1000, 500);

store.width and store.height​​

返回工作区域的宽高。

console.log('宽度为', store.width);

store.scale​​

获取当前画布的缩放值

console.log('zoom is', store.scale);

store.setScale(zoom)​​

设置当前画布的缩放值

store.setScale(3);

UI​

store.openedSidePanel​

获取当前侧边面板的激活状态

备注

默认的侧边面板有：templates、text、materials、background、upload、layers、size、shapes

store.openSidePanel(panelName)​​

手动设置侧边面板的状态

store.openSidePanel('templates');

元素​

store.selectedElements​

返回画布上选中的元素数组

const firstSelected = store.selectedElements[0];

store.selectElements(element)​

设置选择画布上的元素

store.selectElements([element1, element2]);

store.removerElement(element)​

删除画布中选择的元素， element 可传可不传，默认会获取当前聚焦的元素

store.removerElement(element?);

store.getElementById(id)​

通过传递的 id 查找画布中的元素

const element = store.getElementById('id');

console.log(element); // 普通对象，不具备fabric元素实例的方法

store.groupElements()​

组元素，它将创建一个 type 类型为 group 的组元素，不需要传递参数，默认获取画布中聚焦元素来打组。

store.groupElements();

store.ungroupElements()​

取消元素的分组。它将把组内的所有元素移动到页面并删除该组，不需要传递参数，默认获取画布聚焦 type 类型为 group 的组元素来分组。

store.ungroupElements();

历史记录​

store.canUndo​

查看是否有撤销

store.canRedo​

查看是否有重做

store.history.undo()​

撤销回上一次修改

store.history.undo();

store.history.redo()​

重做更改

// 取消更改
store.history.undo();

// 再次应用更改
store.history.redo();

store.history.initialize()​​

清除历史记录

序列化​

store.toJSON()​​

将画布中的元素，导出为对象数组

const json = store.toJSON();

// 存储JSON到某处
saveIntoBackend(JSON.stringify(json));

store.loadJSON(json, keepHistory?)​​

将传递的 json 加载到存储中。它将更新所有属性、页面和元素。默认情况下，将 JSON 加载到存储中将重置撤消/重做历史记录（因此用户无法撤消它）。如果您想保留历史记录，可以使用 keepHistory 参数。

import { createStore } from 'zhihui-editor/lib/model/store';
const store = createStore({
  token: YOUR_TOKEN // 联系我们智绘助理获取: https://liaojunhao.github.io/zhihui-editor/pricing
});

// 在某个时间点保存为JSON
const json = store.toJSON();

// 加载 JSON
// 注意 json 是个 javascript 对象
// 如果您有一个 json 字符串，则可能需要解析它 - JSON.parse(string);
store.loadJSON(json);

// 清空历史记录后来加载 JSON
store.loadJSON(json, true);

事件​

store.on('change', () => {}) ​

监听 store 中的任何变化。该事件可能会在某些操作（例如调整大小或拖动）时频繁触发。store.on() 将返回取消订阅功能。

// 取消监听功能
const off = store.on('change', () => {
  console.log('something changed');
});
// 该行将取消监听该事件
off();

// 编写一个节流函数
let timeout = null;
const requestSave = () => {
  // 如果已请求保存 - 则不执行任何操作
  if (timeout) {
    return;
  }
  // 保存数据到后端
  timeout = setTimeout(() => {
    // 重置超时
    timeout = null;
    // 导出JSON数据
    const json = store.toJSON();
    // 报错数据到后台
    fetch('https://example.com/designs', {
      method: 'POST',
      body: JSON.stringify(json)
    });
  }, 1000);
};

// 请求对任何更改进行保存操作
store.on('change', () => {
  requestSave();
});

导出​

await store.saveAsImage()​​

saveAsImage 将绘图导出为图像并将其保存为本地文件。

// 默认导出
store.saveAsImage({ format: 'png', quality: 1, multiplier: 1 });

// 导出 jpg
store.saveAsImage({ format: 'jpeg', quality: 1, multiplier: 1 });

// 使导出的图像放大 2 倍
store.saveAsImage({ format: 'png', quality: 1, multiplier: 2 });

// 控制大小跟质量（质量只有jpeg格式才生效，png默认已最高质量）
store.saveAsImage({ format: 'jpeg', quality: 0.5, multiplier: 2 });

// 自定义名称
store.saveAsImage({ fileName: 'zhihui-editor.png', format: 'png', quality: 1, multiplier: 1 });

await store.toDataURL()​​​

将存储导出为 Base64 URL 格式的图像

await store.toBlob()​​

将存储导出到 blob 对象，它可能比 toDataURL 方法性能更优

字体​

store.addFont({ fontFamily, src, img })​​

添加新自定义字体的功能。添加的字体将保存到文字侧面板中。如果您想添加全局可用的字体，请查看字体教程。

store.addFont({
  fontFamily: 'MyCustomFont', // 字体名字
  url: serverUrlOrBase64, // 字体链接
  img: 'MyCustomFontImg' // 字体预览图
});

或者您可以直接在 store 中注册字体，然后手动将所需的 字体CSS 添加到页面中：

store.addFont({
  fontFamily: 'MyCustomFont'
});

store.removeFont(name)​​​

按名称从 store 中删除自定义字体

store.removeFont('MyCustomFont');

store.loadFont(name)​​

加载自定义的远程字体，但如果您想要在 UI 的某个地方显示字体列表，它会很有用。store.loadFont(name)​ 函数会向网页添加字体，以便浏览器可以呈现正确的字体。

备注

我们网页使用的字体都是通过 @font-face 的CSS @规则添加的，它允许网页开发者为其网页指定在线字体。通过这种作者自备字体的方式，@font-face 可以消除对用户电脑字体的依赖。 @font-face 不仅可以放在在 CSS 的最顶层，也可以放在 @规则 的 条件规则组 中。

@font-face {
  font-family: 'Open Sans';
  src:
    url('/fonts/OpenSans-Regular-webfont.woff2') format('woff2'),
    url('/fonts/OpenSans-Regular-webfont.woff') format('woff');
}
